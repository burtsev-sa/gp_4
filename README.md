# ГП4

### **Подготовка**

0. Склонировать репозиторий

```bash
git clone https://github.com/burtsev-sa/gp_4
```

1. Создать и активировать окружение

```bash
python3 -m venv .venv
source .venv/bin/activate
```

2. Установить требуемые библиотеки

```bash
pip3 install -r requirements.txt
```

Проект состоит из двух частей:

## Часть 1

Работа над этой частью проекта была проведена в `part_one.ipynb`

## Часть 2

Для реализации этой части за основу был взят датасет [Hotel booking demand](https://www.kaggle.com/code/omniayasser2512/hotel-booking-demand). Это датасет о различной информации по бронированиям в двух отелях: City Hotel и Resort Hotel.

### Основная бизнес-задача

В качестве целевой переменной в датасете используется признак, определяющий отменена ли была бронь, соответственно мы ставим перед собой задачу понять, из-за чего гости отменяют свои брони, в каких отелях это проявляется чаще, что можно сделать и на что обратить внимание, чтобы минимизировать отмены.

### Подготовка и обзор

Перед тем как глубже погрузиться в датасет, провести полноценный EDA и сформировать гипотезы - был проведен первичный анализ, удалены дубликаты, обработаны пропуски, отдельно просмотрен каждый признак, из чего были выделены самые основные, которые потенциально могли бы быть полезными для нас.

Эта часть сделана в `overview_and_preparing.ipynb`

### EDA и гипотезы

Основной содержательный блок работы над частью 2 был сделан в `main_eda.ipynb`. Здесь были детальнее рассмотрены взаимодействия между признаками, посмотрели что влияет на таргет, сформулировали гипотезы и проверили их.
